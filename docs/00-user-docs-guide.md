# ユーザードキュメント規範

> Status: Active
> 最終更新: YYYY-MM-DD

本ドキュメントは、プロジェクトの利用者向けドキュメント（使い方、FAQ、トラブルシュート等）の書き方を定義する。
設計ドキュメント（`design/`）とは目的が異なるため、許容する表現・情報の粒度を分ける。

---

## 適用範囲

本規範は `docs/` ディレクトリ配下に適用する。
構造・メタ情報は[フォーマット規範](../design/00-format-guide.md)に従う。

---

## 基本原則

| 原則 | 内容 |
|------|------|
| **Taskファースト** | 読者が「今やりたいこと」を最短で達成できる構成にする |
| **コピペ可能** | コマンドや設定例はそのまま実行できる形で示す |
| **前提を明示** | OS、権限、依存ツール、権限（管理者/一般）を冒頭に書く |
| **失敗を先に書く** | つまずきやすい点は手順の直後に書く |

---

## コード・コマンドの扱い

設計ドキュメントと異なり、ユーザードキュメントではコードブロックを積極的に使用してよい。
ただし、以下を満たす。

| 項目 | ルール |
|------|--------|
| 実行前提 | どこで実行するか（ローカル/サーバー/コンテナ）を明記する |
| 影響範囲 | 破壊的操作（削除、上書き、課金）には警告を付ける |
| 出力例 | 重要な確認点がある場合のみ最小限の出力例を載せる |

---

## ドキュメント種別（推奨）

| 種別 | 目的 | 例 |
|------|------|----|
| Quickstart | 最短で動かす | インストール、初期設定、初回操作 |
| How-to | タスクを達成する | 追加/変更/移行/復旧など |
| Reference | 正確な仕様を引く | 設定項目、コマンド、エラーコード |
| FAQ | 問い合わせを減らす | よくある質問と回答 |
| Troubleshooting | 復旧を早める | 症状→原因→対処の一覧 |
